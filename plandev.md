# 🌌 แผนการพัฒนาและวิเคราะห์ระบบจัดการข้อมูลเวชภัณฑ์ยา RxMuk

เอกสารฉบับนี้วิเคราะห์ผลกระทบจากการอัปโหลดข้อมูล สรุปแนวทางการออกแบบสถาปัตยกรรมระบบ และข้อเสนอแนะรูปแบบรายงานที่เหมาะสำหรับการใช้งานระดับจังหวัดมุกดาหาร

---

## 1. 🔍 การตรวจสอบเรื่องภาษาไทยเพี้ยนและรหัสวิทยาศาสตร์ (Scientific Notation)

### 1.1 ปัญหาภาษาไทยเพี้ยน (`à¹'à¸¡à¹'à¸¡à¹`)
* **สถานะปัจจุบัน**: **แก้ไขสำเร็จแล้ว 100%**
* **คำอธิบาย**: บั๊กเดิมเกิดขึ้นในฟังก์ชันตรวจจับรหัสภาษา (Encoding Check) ที่มีเงื่อนไข `text.includes('')` ซึ่งมีค่าเป็นจริงเสมอกับทุกข้อความ ส่งผลให้เบราว์เซอร์บังคับแปลงไฟล์ที่เป็น UTF-8 ไปอ่านแบบ Windows-874 (TIS-620) ทำให้อักษรภาษาไทยกลายเป็นตัวต่างดาว 
* **แนวทางแก้ไข**: ปรับการอ่านไฟล์ CSV มาใช้การโหลดแบบ `ArrayBuffer` และพยายามถอดรหัสด้วย `TextDecoder('utf-8', { fatal: true })` เป็นลำดับแรก หากผิดพลาด (เนื่องจากไฟล์จริงเป็น TIS-620/Windows-874) ระบบจะตรวจพบและสลับไปถอดรหัสด้วย `windows-874` โดยอัตโนมัติ ทำให้ภาษาไทยแสดงผลได้อย่างถูกต้องทุกกรณี

### 1.2 ปัญหารหัสยามาตรฐานกลายเป็นวิทยาศาสตร์ (`1.05573e+23`)
* **สถานะปัจจุบัน**: **แก้ไขสำเร็จแล้ว 100%**
* **คำอธิบาย**: ตัวอ่าน SheetJS ในอดีตแปลงค่าคอลัมน์ที่เป็นตัวเลขล้วน (เช่น รหัสยา 24 หลัก หรือรหัสสถานบริการ) ให้กลายเป็น `Number` ใน JavaScript ทำให้ตัวเลขที่ยาวเกิน 16 หลักสูญเสียความละเอียดและถูกปัดกลายเป็นรหัสวิทยาศาสตร์
* **แนวทางแก้ไข**:
  1. สำหรับไฟล์ CSV: บังคับให้อ่านเป็นข้อความดิบ (Raw String) เสมอผ่านตัววิเคราะห์ตัวอักษรโดยตรง
  2. สำหรับไฟล์ Excel: ส่งค่า `{raw: false, defval: ''}` ไปยังฟังก์ชัน `sheet_to_json` ของ SheetJS เพื่อดึงผลลัพธ์เป็นค่าข้อความบนเซลล์แทนที่จะดึงเป็นค่าตัวเลขดิบ
  3. ครอบฟังก์ชัน `String(row[didstdKey]).trim()` ขณะจับคู่คอลัมน์เพื่อให้แน่ใจว่าค่าจะถูกประมวลผลต่อในรูปแบบ String เท่านั้น

### 1.3 ปัญหารหัสหน่วยบริการ รพ.สต. ขาดเลข 0 นำหน้า (เช่น `5777` แทนที่จะเป็น `05777`)
* **สถานะปัจจุบัน**: **แก้ไขสำเร็จแล้ว 100%**
* **คำอธิบาย**: รหัสสถานบริการของ รพ.สต. มักเริ่มต้นด้วยเลข `0` (เช่น `05777`) แต่เมื่อไฟล์ CSV หรือ Excel ถูกบันทึก/ส่งออกผ่านโปรแกรมสเปรดชีต ตัวเลขที่มีเลขศูนย์นำหน้าจะถูกตัดกลายเป็นเลข 4 หลัก (`5777`) ทำให้ระบบกรองฐานข้อมูล `hospitals_db.js` ที่ใช้คีย์ 5 หลัก (`05777`) ไม่พบคู่สถานบริการและไม่สามารถดึงชื่อ รพ.สต. มาแสดงผลได้ (แสดงเป็น "หน่วยบริการ 5777")
* **แนวทางแก้ไข**: ในส่วนกระบวนการแยกวิเคราะห์ข้อมูล ได้เพิ่มตัวกรองการฟอร์แมตรหัสสถานบริการ (`hospcode`) โดยตรวจจับหากรหัสที่แปลงเป็นข้อความมีขนาด 4 หลัก จะทำการเติมตัวเลข `'0'` นำหน้าให้เป็น 5 หลักอัตโนมัติ พร้อมตัดจุดทศนิยมส่วนเกินที่เกิดจากตัวเลขทศนิยมในตาราง (เช่น `5777.0` -> `05777`) ส่งผลให้ระบบดึงชื่อ รพ.สต. มุกดาหาร ได้อย่างถูกต้องสมบูรณ์

## 2. 📊 วิเคราะห์โครงสร้างข้อมูลและสถาปัตยกรรมที่เหมาะสม (System Design)

เปรียบเทียบไฟล์ข้อมูลสองประเภทที่คุณเตรียมนำเข้าจังหวัดมุกดาหาร:

| คุณลักษณะ | 1. `s_tmp_drug_opd.csv` (Summary) | 2. `tmp_drug_opd.csv` (Record-level) |
| :--- | :--- | :--- |
| **ขนาดไฟล์** | **4,004 KB (ประมาณ 4 MB)** | **532,841 KB (ประมาณ 532 MB)** |
| **จำนวนแถว** | ประมาณ 43,000 แถว | คาดว่ามากกว่า 2,000,000 แถว |
| **โครงสร้างภายใน** | เป็นข้อมูลสรุปสะสมระดับหน่วยบริการรายเดือน:<br>`HOSPCODE`, `AMPHUR`, `DIDSTD`, `DNAME`, `SumAmount`, `Count`, `SumDrugCost`, `SumDrugPrice` | เป็นข้อมูลธุรกรรมการจ่ายยารายคนและรายใบสั่งยา:<br>`HN`, `DATE`, `DIDSTD`, `QTY`, `PRICE`, `COST` ฯลฯ |
| **พฤติกรรมบนเว็บ** | **ประมวลผลบนเบราว์เซอร์ได้ทันที** (ใช้เวลาต่ำกว่า 100 มิลลิวินาที)<br>แสดงผลสถิติภาพรวม ยอดซื้อ ยอดสะสม ได้อย่างคล่องตัว | **ไม่สามารถรันบนเบราว์เซอร์โดยตรงได้** (หน่วยความจำเบราว์เซอร์จะล้นและระบบค้างทันทีเนื่องจากขีดจำกัดของ V8 Engine ใน Chrome) |

### 🛠️ ข้อเสนอแนะเชิงสถาปัตยกรรม (Hybrid Data Pipeline)
1. **การประมวลผลบนเครื่องเซิร์ฟเวอร์ (Local Server Process)**:
   * ไฟล์ระดับเรคคอร์ด 532MB (`tmp_drug_opd.csv`) ควรใช้วิธีนำเข้าผ่านสคริปต์หลังบ้าน (PHP CLI หรือ DuckDB / Python) เพื่อทำการคัดกรอง ล้างข้อมูลซ้ำซ้อน และประมวลผลผลลัพธ์ย่อยลงฐานข้อมูลระบบ
   * สั่งให้ระบบเซิร์ฟเวอร์ส่งออก (Export) ข้อมูลรวมยอดรายเดือนในระดับหน่วยบริการเป็นไฟล์ CSV สรุปขนาดเล็ก (คล้ายกับ `s_tmp_drug_opd.csv`)
2. **การประมวลผลแดชบอร์ดแดร็กแอนด์ดรอป (Web Frontend)**:
   * หน้าเว็บ RxMuk ในปัจจุบันจะทำหน้าที่รับไฟล์สรุปยอด (`s_tmp_drug_opd.csv` ขนาด 4MB) เพื่อให้ผู้ใช้สามารถวิเคราะห์ เปรียบเทียบ และแสดงผลสรุปในรูปแบบแผนภูมิและตารางกรองได้อย่างรวดเร็วโดยไม่ต้องรอโหลดข้อมูลปริมาณมหาศาล

## 3. 🎯 การออกแบบรายงานที่ตอบโจทย์ผู้ใช้งาน (Report Designs)

เพื่อให้ผู้ใช้งานและผู้บริหารสาธารณสุขจังหวัดมุกดาหารได้รับประโยชน์สูงสุด แนะนำให้พัฒนาหน้าจอรายงานเพิ่มเติม 4 ส่วนหลักดังนี้:

### 3.1 รายงานการเปรียบเทียบราคายาของจังหวัด (Provincial Price Variance Report)
* **รูปแบบรายงาน**: ตารางแสดงรายการตัวยามาตรฐาน (`didstd` และ `dname`) เปรียบเทียบราคาซื้อต่อหน่วย (`price / amount`) ของแต่ละโรงพยาบาลและ รพ.สต.
* **สิ่งที่ผู้ใช้ได้รับ**:
  * ตัวชี้วัดสีแดง-เขียวเมื่อพบบางหน่วยบริการซื้อยาชนิดเดียวกันในราคาที่สูงหรือต่ำกว่าราคาเฉลี่ยของจังหวัดอย่างผิดปกติ
  * ช่วยกำหนดราคากลางจังหวัดและเพิ่มอำนาจในการจัดซื้อจัดจ้างร่วมกัน

### 3.2 แดชบอร์ดตรวจสอบงบประมาณตามเกณฑ์ ABC-VED Matrix
* **รูปแบบรายงาน**: แผนภูมิแกนร่วมที่จัดจำแนกกลุ่มยา:
  * **ABC**: จัดกลุ่มตามมูลค่าซื้อสะสม (A = มูลค่าสะสมสูง 70-80% แรก, B = ปานกลาง, C = ต่ำสุด)
  * **VED**: จัดกลุ่มตามความสำคัญทางคลินิก (Vital = ยาช่วยชีวิตขาดไม่ได้, Essential = ยาจำเป็นพื้นฐาน, Desirable = ยาเสริมประสิทธิภาพอื่น)
* **สิ่งที่ผู้ใช้ได้รับ**:
  * ทราบทันทีว่างบประมาณของหน่วยบริการสูญเสียไปกับยากลุ่มที่มีมูลค่าสูงแต่มีความจำเป็นต่ำ (เช่น ยากลุ่ม A-D) หรือไม่ เพื่อวางมาตรการควบคุมสั่งใช้ยา

### 3.3 รายงานประเมินคุณภาพมาตรฐานข้อมูล (Data Quality Scorecard แยกรายโรงพยาบาล)
* **รูปแบบรายงาน**: ตารางสรุปคุณภาพข้อมูลของแต่ละหน่วยบริการจำแนกเป็นเปอร์เซ็นต์คะแนน:
  * จำนวนแถวที่ไม่มีรหัสยามาตรฐาน (Missing DIDSTD)
  * จำนวนแถวที่รหัสยาไม่ครบ 24 หลัก หรือขึ้นต้นผิดรหัสมาตรฐาน
  * ปริมาณการจ่ายยาที่มีชื่อยาสะกดผิดพลาดเด่นชัด
* **สิ่งที่ผู้ใช้ได้รับ**:
  * รายงานผลงานการส่งข้อมูลเพื่อส่งผู้รับผิดชอบระบบข้อมูล 43 แฟ้มนำไปปรับปรุงแก้ไขที่ฝั่ง HosXP/JHCIS ต่อไป

### 3.4 แผนที่การจ่ายยาจำแนกรายอำเภอ (Geospatial Drug Distribution Map)
* **รูปแบบรายงาน**: แผนที่ GIS ของทั้ง 7 อำเภอในมุกดาหาร แสดงสัดส่วนมูลค่าการจ่ายยาในกลุ่มโรคสำคัญ (เช่น ยาเบาหวาน ยาลดไขมัน ความดันโลหิตสูง)
* **สิ่งที่ผู้ใช้ได้รับ**:
  * แสดงความครอบคลุมของการเข้าถึงระบบสาธารณสุขและแนวโน้มการกระจายตัวของโรคติดต่อเรื้อรัง (NCDs) ทั่วทั้งจังหวัดมุกดาหาร

## 4. 🚀 การปรับปรุงล่าสุด (Latest Updates)

### 4.1 ระบบเลือกช่วงวันแบบยืดหยุ่น (Date Range Filter)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: เพิ่มช่องสำหรับกรอกช่วงวันที่ (`เริ่มวันที่` ถึง `สิ้นสุดวันที่` ในรูปแบบ YYYY-MM-DD ผ่าน HTML5 Date Picker) ในแถวฟิลเตอร์ของตารางสรุปข้อมูลทั้ง 3 ส่วน (รายยา, รายอำเภอ, รายหน่วยบริการ)
* **การทำงาน**:
  1. กรองข้อมูลธุรกรรมจำลองและข้อมูลที่ผู้ใช้นำเข้าโดยยึดตามวันที่จ่ายจริง (`DATE_SERV`) เพื่อให้ผู้ใช้เลือกวิเคราะห์ช่วงวันที่ที่เจาะจงได้ (เช่น เฉพาะสัปดาห์ หรือเฉพาะเทศกาล)
  2. ทำการเชื่อมโยงข้อมูลแบบ Cascade เพื่อให้การระบุช่วงวันที่ในหน้าหนึ่ง จะสอดคล้องและอัปเดตฟิลเตอร์ช่วงวันที่ในแท็บอื่นๆ โดยอัตโนมัติ

### 4.2 แผงกำหนดรอบข้อมูลสำหรับไฟล์สรุป (Ingestion Cycle Date Settings)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: แก้ไขปัญหาที่ผู้นำเข้าอัปโหลดไฟล์สรุปยอด (`s_tmp_drug_opd.csv`) ซึ่งไม่มีคอลัมน์วันที่อยู่ในเนื้อหาไฟล์โดยตรง ส่งผลให้ระบบในอดีตต้องสุ่มหรือสลับไปใช้ค่าดีฟอลต์ (June 2026 / มิถุนายน 2569) เพียงช่วงเวลาเดียว
* **การทำงาน**:
  1. เพิ่มตัวเลือกปี (ค.ศ.) และ เดือน ในหน้าอัปโหลดไฟล์นำเข้า (Import Panel) เพื่อให้ผู้ใช้สามารถระบุ "รอบช่วงเวลาข้อมูล" ก่อนอัปโหลดไฟล์ได้
  2. ระบบแยกวิเคราะห์จะสลับไปดึงข้อมูลปี/เดือนที่ถูกกำหนดจากตัวเลือกนี้มาเป็นวันที่ของธุรกรรมทั้งหมดทันทีเมื่อพิจารณาแล้วว่าไฟล์นั้นๆ ไม่มีคอลัมน์ข้อมูลวันที่ ช่วยแก้ปัญหาระบบฟิลเตอร์แสดงผลไม่ครบถ้วน

### 4.3 การยกระดับความปลอดภัย (Security Hardening)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: ลบปุ่มล็อกอินทดสอบอัตโนมัติ (💡 คลิกเพื่อใส่รหัสสำหรับทดสอบ admin / 123456) ออกจากหน้าหลักและหน้าจอล็อกของรายงานทั้งหมด เพื่อป้องกันการเข้าถึงข้อมูลและรักษาระบบให้ปลอดภัย

### 4.4 ระบบแสดงกราฟวิเคราะห์ยอดจัดซื้อแบบไดนามิก (Interactive SVG Charts)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: เพิ่มกราฟจำลองแท่ง SVG (SVG Bar Chart) ในหน้าสรุปข้อมูลทั้ง 3 ส่วนหลักเพื่อให้ผู้ใช้วิเคราะห์ข้อมูลแบบภาพรวมได้ทันที:
  1. **หน้าสรุปรายตัวยา**: กราฟแท่งแสดง 10 อันดับยาที่มีมูลค่าจัดซื้อสะสมสูงสุด (พร้อมเปอร์เซ็นต์สัดส่วนสะสม) ด้วยดีไซน์สี Purple Gradient สุดพรีเมียม
  2. **หน้าสรุปรายหน่วยบริการ**: กราฟแท่งแสดง 10 อันดับสถานบริการที่มีมูลค่าจัดซื้อสะสมสูงสุด (พร้อมเปอร์เซ็นต์สัดส่วนสะสม) ด้วยดีไซน์สี Emerald Gradient
  3. **หน้าสรุปรายอำเภอ**: กราฟแท่งแสดงสัดส่วนมูลค่ายาตามอำเภอ (คงเดิมพร้อมปรับแต่งการทำงานสอดคล้องกับฟิลเตอร์แบบ Real-time)
* **การทำงาน**: กราฟทุกตัวจะทำการคำนวณและปรับสเกลขนาดแท่งและเปอร์เซ็นต์แบบ Dynamic อัตโนมัติตามตัวกรอง วันที่/ช่วงเวลา/อำเภอ/หน่วยบริการ ที่ผู้ใช้เลือก

### 4.5 ระบบส่งออกข้อมูลรองรับ Excel / CSV ครบวงจร (Multi-Format Export via SheetJS)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: อำนวยความสะดวกให้ผู้ใช้สามารถส่งออกรายงานและข้อมูลดิบได้ทันทีผ่าน 2 รูปแบบไฟล์:
  1. **ปุ่มส่งออก Excel (XLSX)**: ใช้พลังของ SheetJS (`xlsx.full.min.js`) ในการจัดโครงสร้างตารางข้อมูลเป็นไฟล์สเปรดชีต Excel จริง ซึ่งเปิดใช้งานบน Microsoft Excel ของผู้ใช้ได้ทันทีโดยไม่มีปัญหาเรื่องการเข้ารหัสภาษาไทย
  2. **ปุ่มส่งออก CSV**: ส่งออกไฟล์ CSV ด้วยการแนบ UTF-8 BOM สำหรับเปิดอ่านภาษาไทยอย่างถูกต้อง
* **ปุ่มกดที่เพิ่มเข้ามา**: เพิ่มปุ่มสีเขียวพรีเมียม `🟢 ส่งออก Excel` ในตารางสรุปยา, ตารางสรุปอำเภอ, ตารางสรุปหน่วยบริการ, และหน้าต่างตรวจสอบข้อมูลดิบ (Raw Data Preview Modal)

### 4.6 ระบบตรวจหาและโหลดข้อมูลจริงอัตโนมัติเมื่อเริ่มระบบ (Auto-loading of Real CSV Data)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: แก้ปัญหาผู้ใช้อัปโหลดไฟล์ `s_tmp_drug_opd.csv` และ `tmp_drug_opd.csv` จริงเข้าโฟลเดอร์ `uploads/` บนเซิร์ฟเวอร์เรียบร้อยแล้วแต่หน้าเว็บเริ่มต้นยังแสดงผลเป็น Mock Data:
  1. เพิ่มฟังก์ชันตรวจหาไฟล์ `uploads/s_tmp_drug_opd.csv` อัตโนมัติเมื่อดาวน์โหลดหน้าเว็บ
  2. หากเซิร์ฟเวอร์พบไฟล์ ระบบจะทำการดึงข้อมูลผ่าน fetch API, แปลงการเข้ารหัส (รองรับทั้ง UTF-8 และ Windows-874), แยกวิเคราะห์ (CSV Parsing) และนำเข้าข้อมูลจริง **42,972 แถว** (696 ตัวยาจริงของ รพ.มุกดาหาร) เป็นค่าเริ่มต้นของแผงควบคุมระบบโดยอัตโนมัติทันที
  3. บันทึกข้อมูลและประวัติการนำเข้าอัตโนมัติเป็น "System (Auto)" เพื่อให้ตรวจสอบได้ง่าย
  4. หากไม่มีไฟล์จริงในโฟลเดอร์ ระบบจะสลับไปใช้ Mock Data อัตโนมัติเพื่อให้โปรแกรมสามารถรันได้แบบออฟไลน์อย่างปลอดภัย

## 5. 📊 การบูรณาการระบบวิเคราะห์ขั้นสูงและระบบส่งออก Excel ครบทุกมุมมอง (Finalized Advanced Analytics & Excel Exports)

### 5.1 ระบบรายงานขั้นสูงทั้ง 4 แท็บวิเคราะห์ใหม่ (Price Variance, ABC-VED Matrix, Data Quality Scorecard, Geospatial Map)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์ 100% (Production-Ready)**
* **คำอธิบาย**: ระบบสามารถเชื่อมต่อข้อมูล 43 แฟ้มกับแท็บรายงานทั้ง 4 ส่วนหลักได้อย่างมีประสิทธิภาพ:
  1. **Price Variance**: คำนวณความผันผวนราคายา ค้นหารายการซื้อที่ราคาสูง/ต่ำกว่าค่าเฉลี่ย
  2. **ABC-VED Matrix**: จัดคลาสวิเคราะห์ 3x3 ด้วยเกณฑ์มูลค่าจัดซื้อ (ABC) และความจำเป็นทางคลินิก (VED) พร้อมสร้างกราฟโดนัท SVG แสดงสัดส่วน Category I, II, III
  3. **Data Quality Scorecard**: ประเมินคุณภาพข้อมูลแยกหน่วยบริการ (Missing DIDSTD, Invalid DIDSTD, Zero Price) พร้อมคำนวณเกรดและคะแนนเปอร์เซ็นต์อัตโนมัติ
  4. **Geospatial Map**: แสดงแผนที่อำเภอ 7 อำเภอแบบ interactive ไวต่อตัวกรอง ดึงข้อมูล top drugs และหน่วยบริการในพื้นที่แบบ real-time
* **ความปลอดภัยของแท็บ**: ระบบห่อหุ้มตารางและข้อมูลดิบในแท็บเหล่านี้ทั้งหมดภายใต้สิทธิการเข้าถึงของแผงควบคุม (`secured-content`)

### 5.2 การซิงโครไนซ์ตัวกรองและการประมวลผลข้อมูล (Synchronized Filters & Rendering Pipeline)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: ตัวกรองทุกประเภท (ปี, ช่วงเดือน, วันเริ่มต้น-วันสิ้นสุด, อำเภอ, หน่วยบริการ, และรายการยา) ในทุกแท็บรายงานเชื่อมโยงกันอย่างสมบูรณ์ผ่านตัวประมวลผลศูนย์กลาง `handleFilterChange()` เมื่อผู้ใช้เปลี่ยนแปลงตัวกรองในแท็บใดแท็บหนึ่ง ระบบจะอัปเดตสถานะและกระตุ้นการประมวลผลเรนเดอร์ (`render...`) ในแท็บวิเคราะห์อื่นๆ แบบทันทีทันใด (Real-time Cascade Updates)

### 5.3 ปุ่มส่งออก Excel รองรับครบทุกแท็บวิเคราะห์ (Fully Functional Excel Exports)
* **สถานะการพัฒนา**: **เสร็จสมบูรณ์**
* **คำอธิบาย**: เพิ่มประสิทธิภาพการส่งออกไฟล์ข้อมูลวิเคราะห์ผ่านปุ่ม `🟢 ส่งออก Excel` ครบทุกรายงานใหม่ ได้แก่:
  1. **ABC-VED Matrix**: ฟังก์ชัน `exportAbcVedExcel()` ส่งออกรายงานยาจำแนกตามมูลค่าและระดับการควบคุมจัดซื้อ
  2. **Data Quality Scorecard**: ฟังก์ชัน `exportQualityExcel()` ส่งออกรายการวิเคราะห์และประเมินผลคะแนนคุณภาพข้อมูลหน่วยบริการ
  3. **Geospatial Map**: ฟังก์ชัน `exportMapExcel()` ส่งออกตารางสรุปสถิติมูลค่าจัดซื้อยารวมแยกรายอำเภอในจังหวัดมุกดาหาร
* **ความถูกต้อง**: ปลอดภัยจากปัญหาภาษาไทยเพี้ยนและรหัสวิทยาศาสตร์ด้วยการแปลงข้อมูลเข้า Excel worksheet โดยตรงผ่านกลไก SheetJS
