🚀 คู่มือการนำระบบขึ้นใช้งาน (GitHub Pages Deployment Guide)

1. การเตรียมข้อมูล (Data Preparation)

  • ส่งออกไฟล์ tmp_drug_opd หรือ s_tmp_drug_opd จากระบบ HDC
  • หากต้องการให้ระบบโหลดข้อมูลอัตโนมัติ ให้นำไฟล์ไปวางในโฟลเดอร์ uploads/s_tmp_drug_opd.csv

2. การเผยแพร่บน GitHub (Online Hosting)

  • อัปโหลดไฟล์ index.html และ hospitals_db.js ขึ้น GitHub Repository
  • ไปที่ Settings > Pages และเลือก Branch "main" เพื่อเปิดใช้งาน GitHub Pages
  • ข้อมูลของคุณจะถูกประมวลผลบน "เบราว์เซอร์ของผู้ใช้" เท่านั้น (Client-side Only) ไม่มีการส่งข้อมูลกลับไปยังเซิร์ฟเวอร์ ปลอดภัยตามหลัก PDPA

📊 แผงวิเคราะห์งบประมาณและลำดับความสำคัญ ABC-VED Matrix

ABC Classification: จำแนกตามมูลค่ายอดจัดซื้อ (A = มูลค่าสูง 70% แรก, B = มูลค่าปานกลาง 20%, C = มูลค่าต่ำ 10% หลัง)
VED Classification: จำแนกตามความสำคัญทางคลินิก (Vital = ยาช่วยชีวิต/ขาดไม่ได้, Essential = ยาจำเป็นพื้นฐาน, Desirable = ยาเสริมประสิทธิภาพ/ทั่วไป)
Category I (AV, AE, AD, BV, CV): วงกลมสีแดง/ส้ม ควรควบคุมจัดเก็บอย่างเข้มงวดและตรวจสอบราคาซื้อเป็นพิเศษ
Category II (BE, BD, CE): วงกลมสีเหลือง ควบคุมจัดเก็บตามเกณฑ์มาตรฐานทั่วไป
Category III (CD): วงกลมสีเขียว ควบคุมจัดเก็บตามรอบและผ่อนปรนการจัดหาได้
