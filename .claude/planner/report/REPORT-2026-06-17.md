# 📋 รายงานการพัฒนาและการจัดส่งระบบ (Deployment Report) - 17 มิถุนายน 2026

## 1. Summary (สรุปภาพรวม)
การแก้ไขระบบ RxMuk Analytics ในรอบการพัฒนานี้เน้นไปที่การแก้ปัญหา **ข้อมูลเวชภัณฑ์สูญหายหลังจาก Logout** และการตั้งค่าระบบ **Automated Deployment ผ่าน GitHub Actions ไปยัง GitHub Pages** ให้ใช้งานได้สมบูรณ์และลดภาระการใช้ Token ให้มากที่สุด

ผลการดำเนินการสำเร็จลุล่วง 100% โดยระบบจัดเก็บข้อมูลใน `localStorage` แทน `sessionStorage` ทำให้ข้อมูลไม่สูญหาย และได้ปรับแก้เส้นทางการ Deploy ในไฟล์ Workflow ของ GitHub Actions ให้ครอบคลุมไฟล์ HTML หลักและโฟลเดอร์อัปโหลดข้อมูลดิบสำหรับการรันแบบ Serverless (Static Site) บน GitHub Pages ได้อย่างสมบูรณ์

## 3. Errors Encountered (ข้อผิดพลาดที่พบและแก้ไข)
* **ปัญหาการเข้าถึงไฟล์ CSV บน GitHub Pages**: 
  * *วิเคราะห์*: เนื่องจากเดิม Workflow ตั้งค่าการอัปโหลดเฉพาะโฟลเดอร์ `public` ทำให้ไม่สามารถเข้าถึงไฟล์ข้อมูลดิบหลัก `uploads/s_tmp_drug_opd.csv` ได้ และทำให้เกิดข้อผิดพลาด 404 เมื่อทำการทดสอบการจำลองแบบ Serverless
  * *การแก้ไข*: ปรับแก้ไฟล์ `deploy.yml` ให้ชี้เป้าหมายไปที่โฟลเดอร์หลักของโปรเจกต์ (`path: .`) เพื่อให้ดึงไฟล์ HTML หลักและไฟล์ CSV ติดตัวไปด้วยตอนสร้าง Artifact
